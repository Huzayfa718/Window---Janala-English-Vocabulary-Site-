Frequently Asked Questions

What are the differences between var, let, and const?



The main difference between var, let, and const is the scope of the variables. The var keyword is function-scoped, while let and const are block-scoped. This means that variables declared with var are hoisted to the top of the function, while variables declared with let and const are hoisted to the top of the block they are declared in. Additionally, variables declared with const cannot be reassigned, while variables declared with let can be reassigned.

What are the difference between map(), forEach(), and filter()



The main difference between map(), forEach(), and filter() is the return value. The map() method returns a new array with the results of calling a provided function on every element in the array. The forEach() method executes a provided function once for each array element. The filter() method creates a new array with all elements that pass the test implemented by the provided function.

Explain arrow functions and how they are different from regular functions.



Arrow functions are a more concise way to write functions in JavaScript. They are different from regular functions in that they do not have their own this, arguments, super, or new.target keywords. This means that arrow functions cannot be used as constructors and do not have access to the arguments object. Additionally, arrow functions do not have a prototype property.

How do JavaScript Promises work ?



JavaScript Promises are objects that represent the eventual completion (or failure) of an asynchronous operation and its resulting value. They are used to handle asynchronous operations in JavaScript, such as fetching data from a server or reading a file. Promises have three states: pending, fulfilled, and rejected. When a promise is pending, it means that the asynchronous operation is still in progress. When a promise is fulfilled, it means that the asynchronous operation has completed successfully. When a promise is rejected, it means that the asynchronous operation has failed.

How closures work in JavaScript ?



Closures are a powerful feature of JavaScript that allow functions to retain access to variables from their parent scope even after the parent function has finished executing. This means that a function can access variables from its parent scope even after the parent function has returned. Closures are created when a function is defined inside another function and the inner function references variables from the outer function's scope. This allows the inner function to "close over" the variables from the outer function and retain access to them.
